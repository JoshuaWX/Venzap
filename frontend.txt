═══════════════════════════════════════════════════════════
                   WHAT IS VENZAP?
═══════════════════════════════════════════════════════════

Venzap is a two-sided chat-commerce platform. Local vendors
(food sellers, grocery stores, pharmacies, fashion vendors)
register on a web dashboard. Customers order from those vendors
entirely through a Telegram bot — no app download needed.
Payments are powered by Payaza (Nigerian fintech).

The product starts with food vendors and expands to all vendor
types. Built for Nigeria. Designed to scale across Africa.

Target users:
  → Vendors: local Nigerian business owners, not very technical,
    need something simple and trustworthy
  → The website: modern, professional, builds trust immediately

  → Text that clips through background images/gradients (background-clip: text)
  → Large typographic moments — headlines that COMMAND the screen
  → Subtle noise texture overlay on dark backgrounds (grain effect)
  → Glassmorphism cards with orange-tinted borders, not plain dark cards
  → A custom cursor: small orange dot that grows on hover
  → Scroll-triggered animations — every section slides in as you scroll
  → The word "Venzap" should always feel like a brand moment, never plain

NEVER DO:
  ❌ Purple gradients on white backgrounds
  ❌ Inter or Roboto as the main font
  ❌ Flat boring cards with no depth
  ❌ Generic hero with stock image background
  ❌ Default shadcn component styling
  ❌ Cookie-cutter layouts (big text left, image right)
  ❌ Predictable section order that every SaaS uses
  ❌ Overusing emojis as design elements

EFFECTS — IMPLEMENT THESE:

  Noise texture:
    Apply a subtle grain overlay to ALL dark backgrounds.
    Use an SVG filter or a pseudo-element with:
    background-image: url("data:image/svg+xml,...")
    opacity: 0.035
    This adds depth and prevents flat digital look.

  Orange glow orbs:
    Large blurred ellipses positioned absolutely behind content.
    background: radial-gradient(ellipse, var(--orange-glow) 0%, transparent 70%)
    Use on hero, CTA sections. Give depth without being distracting.

  Glassmorphism cards:
    background: rgba(14, 14, 26, 0.7)
    backdrop-filter: blur(20px) saturate(180%)
    border: 1px solid rgba(255, 255, 255, 0.07)
    Apply on cards that sit over gradient/glow backgrounds.

  Text gradient mask:
    Apply to large hero headlines and stat numbers:
    background: linear-gradient(135deg, #FB923C 0%, #F97316 40%, #FBBF24 100%)
    -webkit-background-clip: text
    -webkit-text-fill-color: transparent
    background-clip: text

  Orange top-edge glow on cards:
    Cards on dashboard: box-shadow: 0 -1px 0 0 var(--orange-primary)
    on their top border — gives a subtle "lit from above" look.

  Scan line mask effect:
    On the hero section, apply a very subtle repeating-linear-gradient
    with 1px dark lines at 2px intervals, opacity 0.03.
    Creates a slight screen/monitor texture. Very subtle.

ANIMATIONS (use Framer Motion for all):

  Page load: staggered fade-up. Each section element enters with:
    initial: { opacity: 0, y: 24 }
    animate: { opacity: 1, y: 0 }
    transition: { duration: 0.5, ease: [0.25, 0.46, 0.45, 0.94] }
    Stagger delay: 0.08s between children

  Scroll animations (use Framer Motion's useInView):
    Every section triggers on 20% visibility
    Same fade-up pattern, staggered by 0.1s

  Number count-up: animate stat numbers from 0 to final value
    over 1.8 seconds on scroll into view. Use spring physics.

  Hover on interactive cards:
    whileHover: { y: -3, borderColor: "rgba(249,115,22,0.4)" }
    transition: { duration: 0.2 }

  CTA button:
    whileHover: { scale: 1.02, boxShadow: "0 0 30px rgba(249,115,22,0.4)" }
    whileTap: { scale: 0.98 }

  Custom cursor (on desktop only):
    Small 8px orange dot that follows cursor with a spring lag.
    On hover over links/buttons: expands to 32px circle,
    border: 1px solid orange, background: transparent.
    Built with Framer Motion useMotionValue + spring.
